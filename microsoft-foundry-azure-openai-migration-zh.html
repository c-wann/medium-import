從 Azure OpenAI 升級到 Microsoft Foundry：零停機完整遷移指南

如果您目前正在使用 Azure OpenAI Service 建立 AI 應用，微軟的建議是將其升級為 Microsoft Foundry 資源——不是遷移，而是就地升級。升級後，您的現有模型部署、API 金鑰、端點位址和現有應用狀態全部保留，同時解鎖 Foundry 的代理人、RAG、評估、追蹤、護欄等完整功能。本文說明升級流程、注意事項，以及遷移後要啟用的新能力。

升級前：了解 Foundry 與 Azure OpenAI 的差異

Azure OpenAI Service 是純粹的 OpenAI 模型 API 服務；Microsoft Foundry 是在此之上的完整 AI 開發平台。功能差異如下：模型方面，Foundry 在保留所有 OpenAI 模型的同時，新增 Meta Llama、Mistral、DeepSeek-R1、Phi-4、Claude、Grok 等 1,900+ 個第三方模型，透過統一的 Foundry Models API 存取。代理人方面，Foundry 提供 Agents v2（Responses API）和 Agent Service（含工具、記憶體、知識庫整合），取代舊版的 Assistants API。

終端點相容性：升級後，舊版 Azure OpenAI 端點（https://<resource>.openai.azure.com/）繼續有效；新的 Foundry Direct 端點（https://<resource>.services.ai.azure.com/openai/v1）同時啟用。舊版 API 金鑰（Ocp-Apim-Subscription-Key）繼續可用。這保證了現有應用零停機、零程式碼修改完成升級。

升級步驟

步驟一：登入 Azure Portal，找到您的 Azure OpenAI 資源。在資源概觀頁面的頂部，應該會出現「Upgrade to Microsoft Foundry」的升級提示橫幅。點選後，確認升級對話框中的說明，再點「Upgrade」。整個升級過程通常在幾秒鐘內完成。步驟二：升級後，開啟 Foundry Portal，您的資源和所有現有部署應該可見。步驟三：在 Foundry Portal 建立一個「專案（Project）」，取得新格式的 Project Endpoint（https://<resource>.ai.azure.com/api/projects/<project>），後續所有 Foundry 功能透過此端點使用。

如果升級橫幅沒有顯示，可能是因為您的資源位於尚不支援 Foundry 升級的地區。支援地區清單和各地區功能可用性請見Foundry 地區與配額文件。

API 遷移：從 Assistants API 到 Responses API

舊版 Assistants API 的核心概念是：Thread（對話串）、Message（訊息）、Run（執行）、Assistant（助手）。新版 Responses API（Agents v2）重新命名：Conversation（對話）、Item（項目）、Response（回覆）、Agent Version（代理人版本）。這不只是改名——Responses API 提供更豐富的工具整合、更好的串流支援，以及與 Foundry 護欄、評估、追蹤的深度整合。

遷移的主要工作是更新 SDK 版本（azure-ai-projects>=2.0）和調整 API 呼叫路徑。如果您的舊版應用還需要繼續使用 Assistants API，Foundry 在可預見的未來繼續支援 /openai/assistants/ 端點，不需要立即遷移。但新功能只會在 Responses API 上持續開發，長期來看遷移是值得的。完整的 API 對照表請見Agents 概覽文件。

升級後立即可以啟用的新功能

升級完成後，這些功能可以立即使用，無需額外設定：Model Catalog（探索和試用 1,900+ 個模型）、Prompt Optimizer（自動改善代理人指令）、Guardrails & Controls（包含 Content Safety、Prompt Shields 和 Task Adherence）。以下功能需要少量額外配置：Tracing（需要連接 Application Insights 資源）、RAG 知識庫（需要建立 Azure AI Search 資源和向量索引）、Fine-tuning（需要訓練資料集）、Evaluation（需要評估資料集）。

整體評估：對於已在 Azure OpenAI 上運作的生產應用，升級到 Foundry 幾乎是純收益，因為現有功能完全保留，新功能隨時可按需啟用，成本結構不變（帳單仍在模型部署層）。唯一的注意事項是確認 RBAC 配置：升級後，Foundry Portal 的存取由 Azure AI User（開發者最低權限）、Contributor（資源管理）和 Owner（完整控制）角色控制，需要確認現有 IAM 設定符合您組織的最小權限原則。
