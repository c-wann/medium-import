LangChain 整合 Microsoft Foundry：建立複雜 AI 代理人工作流程的完整指南

LangChain 和 LangGraph 是目前最廣泛使用的 AI 應用開發框架，提供豐富的工具整合、記憶體管理和工作流程圖（Graph）抽象。Microsoft Foundry 與 LangChain 的深度整合讓您可以用 langchain-azure-ai 套件，在 LangChain 和 LangGraph 的應用中直接取用 Foundry 的模型、工具、知識庫、代理人服務和追蹤能力，同時保持熟悉的 LangChain 程式設計模式。

安裝與設定

首先確認您已有 Microsoft Foundry 專案（可從 ai.azure.com 建立或升級自 Azure OpenAI 資源）。安裝基礎套件：pip install -U langchain-azure-ai azure-identity。依據您的使用情境，可以追加安裝選擇性功能：pip install "langchain-azure-ai[tools]"（啟用 Document Intelligence 等工具命名空間）或 pip install "langchain-azure-ai[opentelemetry]"（啟用 OpenTelemetry 追蹤整合）。

設定環境變數：export AZURE_AI_PROJECT_ENDPOINT="https://<resource>.services.ai.azure.com/api/projects/<project>"。設定這個端點後，大多數 langchain-azure-ai 類別無需其他設定就能自動完成認證（透過 DefaultAzureCredential）。如果使用 API 金鑰，改設 OPENAI_BASE_URL 和 OPENAI_API_KEY 環境變數。

呼叫 Foundry 模型

有兩種方式在 LangChain 中呼叫 Foundry 模型。方式一（推薦）：使用 LangChain 的通用 init_chat_model，指定 "azure_ai:<model_name>" 前綴，例如 model = init_chat_model("azure_ai:gpt-5")。這讓您在不修改業務邏輯的情況下切換不同的 Foundry 模型。方式二：使用 AzureAIOpenAIApiChatModel 類別，可以明確指定 project_endpoint（使用 Entra ID 認證）或直接的 endpoint（使用 API 金鑰）。這兩種方式都支援 LangChain 標準的 .invoke()、.stream() 和 .batch() 介面。

模型不限於 OpenAI 系列——因為 Foundry 模型目錄統一了所有模型的存取介面，您可以用完全相同的程式碼呼叫 "azure_ai:Meta-Llama-3.3-70B-Instruct"、"azure_ai:mistral-large-2411" 或 "azure_ai:DeepSeek-R1"，在不同模型之間做 A/B 比較，而不需要更改 LangChain 的 Chain 或 Agent 邏輯。

LangGraph + Foundry Agent Service

langchain_azure_ai.agents 命名空間讓您在 LangGraph 圖（Graph）中使用 Foundry 代理人節點。典型的應用模式是：建立多個專職代理人（如「研究代理人」、「撰寫代理人」、「審查代理人」），用 LangGraph 定義這些代理人之間的協作流程（邊和條件判斷），再透過 Foundry 的 Multi-agent 能力讓每個代理人可以彼此呼叫。Foundry Content Safety Middleware 也可以在 LangGraph 的邊上加入護欄，確保代理人之間傳遞的訊息符合安全要求。

LangChain 的記憶體（Memory）整合是另一個亮點。langchain_azure_ai.chat_message_histories 讓您把對話歷史儲存在 Azure Cosmos DB 或 Azure AI 的記憶體功能中，跨 Session 保持對話連續性，適合需要記住用戶偏好或長期對話脈絡的應用。

向量儲存與 RAG

langchain_azure_ai.vectorstores 提供 Azure AI Search 和 Azure Cosmos DB 的向量儲存整合，讓您在 LangChain 的 RAG 鏈（Retrieval Chain）中直接使用 Foundry 的知識庫。langchain_azure_ai.embeddings 提供 Foundry 模型目錄中各 Embedding 模型的 LangChain 介面，包括 text-embedding-3-large、cohere-embed-v3 等。這讓您可以在同一個 LangChain 程式碼中，比較不同 Embedding 模型對 RAG 檢索精準度的影響。

追蹤整合是 langchain-azure-ai 的一大優勢。安裝 [opentelemetry] 額外套件後，langchain_azure_ai.callbacks 會自動把 LangChain 的執行事件以 OpenTelemetry 格式匯出到 Foundry 的 Application Insights。這讓您在 Foundry Portal 的 Tracing 頁面查看 LangChain 和 LangGraph 工作流程的完整執行追蹤，而不需要單獨維護一套觀察性基礎設施。完整的 LangChain 整合文件請見官方 LangChain with Foundry 指南。

遷移現有 LangChain 應用的建議

如果您已有使用 langchain-openai（連接 Azure OpenAI）的現有應用，遷移到 langchain-azure-ai 只需要：替換 import（from langchain_azure_ai.chat_models import ...）、設定新的環境變數（AZURE_AI_PROJECT_ENDPOINT），以及調整模型名稱為 "azure_ai:<model>" 格式。大多數 LangChain 的 Chain、Tool、Memory 和 Callback 邏輯不需要修改。Foundry 的 Serverless 模型部署讓您可以同時測試多個模型，找到成本與效能的最佳平衡後再正式上線。
