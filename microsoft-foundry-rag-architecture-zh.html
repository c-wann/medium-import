Azure AI Foundry RAG 架構完整指南：讓 AI 代理人讀懂您的企業資料

再強的語言模型，一旦問到您公司的內部文件、最新政策或私有資料庫，就會無從回答。Retrieval-Augmented Generation（RAG，檢索增強生成）是解決這個問題最成熟的架構模式，而 Microsoft Foundry 提供了完整的 RAG 工具鏈，從向量索引、語義搜尋到代理人工作流程，一站到位。

RAG 是什麼？解決什麼問題？

傳統 LLM 的知識截止於訓練資料；它不知道昨天的新聞、您公司今早更新的 SOP，或者您資料庫裡的客戶資料。RAG 的核心思想很簡單：每次用戶提問時，先去外部知識庫撈出相關文件片段，把它們塞進 LLM 的上下文（Context），再讓 LLM 根據這些參考資料生成回覆。這樣模型就能「知道」它訓練資料以外的事情，而且回答更具可信度（有來源可查）。

在 Foundry 的架構下，RAG 管線的組成元素包括：文件儲存（SharePoint、Blob Storage、本地檔案）、向量化（Embedding Model）、向量索引（Azure AI Search）、檢索（Retrieval Tool）、以及最終的答案生成（LLM）。每一個環節 Foundry 都有對應的服務整合。

步驟一：建立向量索引

在 Foundry Portal，進入您的專案，選擇左側導覽列的 Knowledge bases（知識庫），點選 Create vector index。您可以選擇資料來源：Azure Blob Storage、Azure AI Search（已有的索引）、SharePoint、OneLake，或直接上傳本地檔案。Foundry 會自動處理文件切塊（Chunking）、向量化（使用您選擇的 Embedding 模型，如 text-embedding-3-large），並建立 Azure AI Search 索引。

切塊策略對 RAG 效果影響很大。Foundry 提供可調整的 Chunk Size（預設 1,024 tokens）與 Overlap（預設 256 tokens）設定。如果您的文件是長篇技術手冊，較大的 chunk size 保留更多上下文；如果是 FAQ 式的短問答，較小的 chunk size 能提升精準度。實務上建議針對您的文件類型分別測試不同設定。

步驟二：把知識庫連接到代理人

建好知識庫後，在代理人（Agent）設定頁面，選擇 Add knowledge，將您的向量索引連接上來。Foundry 代理人預設使用代理人式檢索（Agentic Retrieval）：每次用戶提問，代理人會自動判斷是否需要查知識庫，並根據問題的意圖分解成多個子查詢平行搜尋，最後合成有依據的回覆。這種方式比簡單的「向量相似度搜尋 + 填充上下文」更智慧，尤其適合複雜的多跳查詢（Multi-hop queries）。

如果需要更精細的控制，您也可以在程式碼中直接使用 Foundry SDK 的 AzureAISearchTool：

from azure.ai.agents.models import AzureAISearchTool, AzureAISearchQueryType — 設定 index_connection_id、index_name 與 query_type（支援 SIMPLE、SEMANTIC、VECTOR、VECTOR_SEMANTIC_HYBRID），即可讓代理人使用您的 Azure AI Search 索引。完整範例請見官方文件。

混合搜尋：關鍵字 + 語義向量

單純的向量相似度搜尋有時會漏掉精確的詞彙匹配（例如產品編號、人名），而單純的關鍵字搜尋又缺乏語義理解能力。Foundry 推薦使用 VECTOR_SEMANTIC_HYBRID 查詢模式，結合向量搜尋、BM25 關鍵字搜尋與語義重排序（Semantic Reranker），在大多數情況下能同時兼顧精準度與語義理解。Azure AI Search 的語義重排序功能是免費的前 1,000 個查詢/月，超過才開始計費。

對於需要跨多個索引或多種資料來源的複雜場景，Foundry 的 Foundry IQ（預覽）功能提供了統一的知識管理層，支援多知識庫路由與結果融合。如果您的企業有大量非結構化文件，建議評估這個進階功能。詳細說明請見官方 RAG 概念文件。

評估 RAG 品質的三個核心指標

RAG 系統的品質評估不能只看「回答對不對」，還需要衡量三個維度：Context Recall（上下文召回率）——知識庫有沒有把相關文件找出來；Groundedness（接地氣性）——模型的回答有沒有忠實反映檢索到的文件，沒有胡亂捏造；Relevance（相關性）——回答是否真正解答了用戶的問題。Foundry 的評估框架提供這三個指標的自動化評分，讓您在迭代索引與提示詞設計時有量化依據。
