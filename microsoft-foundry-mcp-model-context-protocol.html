MCP in Microsoft Foundry: Connect Any Tool to Your AI Agent Without Writing Integration Code

The Model Context Protocol (MCP) is quietly becoming the USB standard of the AI world — a universal interface that lets AI agents connect to tools, data sources, and services regardless of who built them. Microsoft Foundry's first-class MCP support means you can now connect agents to virtually any capability with a few lines of configuration, replacing weeks of custom integration work with a standardized connection pattern.

What MCP Enables in Foundry

In Microsoft Foundry Agent Service, MCP is a first-class custom tool type alongside OpenAPI tools and Agent-to-Agent (A2A) connections. An MCP server is simply an endpoint that exposes tools using the Model Context Protocol. When you connect an MCP server to a Foundry agent, the agent can discover the tools it exposes and call them autonomously during conversations, the same way it would use built-in tools like web search or code interpreter.

The Foundry portal includes a browsable tool catalog with pre-configured remote MCP servers from Microsoft and third-party publishers. The Azure DevOps MCP Server (public preview), for example, can be connected directly from the catalog — you supply your Azure DevOps organization and a subset of permitted tools, and the agent gains access to DevOps data and actions without writing any integration code.

Building Your Own MCP Server on Azure Functions

Custom MCP servers are equally well-supported. Azure Functions has a dedicated MCP webhook endpoint (/runtime/webhooks/mcp) that lets you expose any Azure Function as an MCP-compatible tool. This means your existing business logic — order lookups, CRM queries, inventory checks — can become agent tools by deploying a Functions app and registering the endpoint in the Foundry portal. No special SDK or MCP-specific code is required; Azure Functions handles the protocol translation.

Supported authentication options for MCP server connections include key-based access (API key stored in a project connection), Microsoft Entra authentication via the agent's managed identity (which eliminates the need to manage secrets entirely), and OAuth On-Behalf-Of for user-delegated permissions where the agent acts on behalf of the authenticated end user. Microsoft recommends starting with Entra authentication when supported, since it provides built-in token rotation with no secret management overhead.

Toolbox: One MCP Endpoint for All Your Tools

The new Toolbox feature (currently in public preview) extends the MCP pattern to tool management at scale. Instead of attaching tools individually to every agent definition — duplicating authentication configuration and integration code across dozens of agents — you define a curated set of tools once in a Toolbox, and Foundry exposes it as a single MCP-compatible endpoint.

Any MCP-capable runtime can consume a Toolbox: agents built with Microsoft Agent Framework, LangGraph, GitHub Copilot SDK, or custom code all connect to the same endpoint. Toolbox versioning gives explicit control over rollouts — create a new version, test it against the version-specific endpoint, and promote it to default when ready. Agents consuming the default endpoint automatically receive the updated version without any code changes. Authentication for all tools in the Toolbox is managed centrally, with Foundry handling credential injection and token refresh at runtime using Microsoft Entra and OAuth.

Security and Governance for MCP Connections

MCP's flexibility creates real governance challenges: agents could inadvertently be connected to malicious or poorly-secured servers. Foundry addresses this with structured tool configuration, credential isolation in project connections, and runtime governance through an AI gateway. The gateway lets administrators apply rate limits, IP restrictions, and usage policies to MCP tools at the network level — ensuring that even third-party MCP servers operate within organizational controls.

A private tool catalog lets organizations publish internal MCP servers that only users within their organization can discover and configure — preventing accidental exposure and ensuring the tools available to agents have been vetted by the platform team. For teams building AI systems that need to integrate with complex enterprise tooling landscapes, MCP in Foundry is the practical path from fragmented integrations to a governed, reusable tool ecosystem.
